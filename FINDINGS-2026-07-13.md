## Root causes of the library gap (ranked by leverage)

1. **Canonical lyrics missed on 11/30 songs** (LRCLIB misses on TV cuts,
   remixes, obscure titles) → raw Whisper text → lyric_similarity craters and
   line breaks fall back to silence-gap grouping (e.g. 90 s song → 2 giant
   lines). *Candidate fix: paste-lyrics box in the web UI feeding
   `align.build_words` directly; cheap and bypasses the API entirely.*
2. **Japanese lyric_similarity ≈ 0 is a scoring artifact** — gold charts are
   romaji, Whisper emits kana/kanji. Timing match for ja is real (0.22–0.60).
   *Candidate fix: romanize both sides in `evaluate.py` before comparing (or
   just document it); don't chase it as a generation bug.*
3. **Whisper collapse on processed vocals** — worst case Rihanna Remix:
   24 words in 260 s → match 0.03. *Candidate fix: result-summary warning when
   words/minute is absurdly low, so the user retries with different options
   instead of trusting the draft.*
4. Note-count ratio median 0.81 (under-generation) on the library vs 1.28
   (over-generation) on the benchmark — syllable spread vs gold charting style
   varies; no single fix, revisit after 1–3.
