## Root causes of the library gap (ranked by leverage)

1. **Canonical lyrics missed on 11/30 songs** (LRCLIB misses on TV cuts,
   remixes, obscure titles) → raw Whisper text → lyric_similarity craters and
   line breaks fall back to silence-gap grouping (e.g. 90 s song → 2 giant
   lines). *Candidate fix: paste-lyrics box in the web UI feeding
   `align.build_words` directly; cheap and bypasses the API entirely.*
2. **Japanese lyric_similarity ≈ 0 is a scoring artifact** — gold charts are
   romaji, Whisper emits kana/kanji. Timing match for ja is real (0.22–0.60).
   *Candidate fix: romanize both sides in `evaluate.py` before comparing (or
   just document it); don't chase it as a generation bug.*
3. **Whisper collapse on processed vocals** — worst case Rihanna Remix:
   24 words in 260 s → match 0.03. *Candidate fix: result-summary warning when
   words/minute is absurdly low, so the user retries with different options
   instead of trusting the draft.*
4. Note-count ratio median 0.81 (under-generation) on the library vs 1.28
   (over-generation) on the benchmark — syllable spread vs gold charting style
   varies; no single fix, revisit after 1–3.

- Solo benchmark regression after all changes: match 0.798, pitch corr 0.622 —

### Duet attribution A/B (12 library duets with [MULTI].txt, seed 0)
`batch_eval.py --duets-only --diarize yes|no` (new flags). 12/12 scored in
both legs after the metadata fix. Library has 253 [MULTI] duet folders.

- **Split coverage is the real difference**: diarization split 4 of the 6
  multi-artist-named duets (saa 0.617, 0.637, 0.805, 0.613; median 0.627);
  register clustering split only 2 (0.685, 0.631) — its lopsided-split guard
  bails where diarization succeeds.
- On the two songs both methods split, accuracy is comparable (diar 0.805/0.613
  vs clust 0.685/0.631, one win each). saa stays in the 0.6–0.8 band — the
  shared-stem limit noted on the bundled benchmark holds on the library.
- **Structural gap**: 6/12 sampled duets have single-name artists (Evanescence,
  Glee, Ha-Ash, Onda Vaselina, Aladdín, Spandau Ballet) so `is_multi_artist`
  never attempts a split; 2 more multi-named ones (Yandel ft. Gadiel, Juan
  Fernando Velasco y Fonseca) stayed solo in both legs. **Resolved by product
  decision**: the web UI's mode selector lets the user force the duet split,
  so no name-based auto-detection is needed; forcing duet now also engages
  diarization under `--diarize auto` (previously it fell back to register
  clustering unless the artist name looked multi). Auto mode keeps the
  conservative name gate, protecting the n=30 run's 0-false-splits.
