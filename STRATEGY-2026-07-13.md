## Observed failures (user-reported, on deployed songs)

1. **Chuchaqui** — opens with "Voy", then dead silence, then the verse
   continues. The count-in re-anchor moved "Voy" *near* "con", but every
   word's absolute time is still Whisper's guess; the whole opening is
   imprecise.
2. **Ama De Mi Sol (Milo J)** — elongated syllables overlap the next word,
   corrupting note identification; **and many words are simply missing**.
   Whisper emitted 202 words in 176 s (~69 wpm) for a dense track; our merge
   is Whisper-driven, so every canonical word Whisper skipped is *dropped
   from the chart* (align.py `build_words` keeps only Whisper-timed words).
3. General: timing feel is far from the hand-made charts in
   `D:\Canciones Karaoke` (community-rated, the quality bar).

## Root causes (ranked)

1. **Whisper word timestamps are not measurements.** They come from
   cross-attention DTW; typical error ±100–300 ms, catastrophic at phrase
   starts after instrumental gaps and on elongated vowels (the vowel's mass
   bleeds into the next word). All the snap/re-anchor/gap-guard heuristics
   added on 2026-07-13 are patches over this.
2. **Whisper-driven merge drops canonical words.** `align.build_words` keeps
   Whisper's word list and re-texts it. Low Whisper recall (fast singing,
   heavy processing) ⇒ missing chart words. The canonical lyric should be
   the *unit of output*, not a relabeling source.
3. **Proportional syllable spreading.** Syllables split a word's span by
   character count; a held vowel is nothing like proportional. Needs
   phoneme/char-level acoustic timing.
4. **Fixed 240 "file BPM" grid.** Hand charts use the song's real musical
   BPM, so notes land on musical subdivisions. Ours quantizes to a
   62.5 ms grid with no relation to the beat — feels mechanical even when
   onsets are right. (`tempo.estimate_bpm` already exists, unused.)

## The new perspective: the library is ground truth, not just a style profile

`D:\Canciones Karaoke` = ~5900 (gold `.txt`, `.mp3`) pairs. A gold chart is a
human-verified transcription of *word/syllable onsets, durations, and
relative pitches* for its exact audio file. That is per-stage supervision we
have not used: so far we only compared final output (batch_eval) and pooled
style stats (library_profile).

### Phase 0 — stage-level eval harness (`library_replay.py`, build first)

For a stratified sample (~100 songs; by language and by wpm density), parse
the gold chart into time-domain word/syllable events, then score each
pipeline stage **independently** against it:

- **Whisper timing**: for words matched gold↔whisper (text match), onset and
  end error distributions; word recall (fraction of gold words Whisper
  emitted at all). This answers "tan mal es Whisper?" with numbers, per
  language and per tempo.
- **Forced alignment timing** (candidate replacement, see Phase 1): same
  metrics for MMS_FA CTC alignment of the gold lyric text against the
  separated stem. Decision gate: adopt whichever wins per-stage.
- **Syllable durations**: proportional spread vs char-span cuts vs gold.
- **Pitch**: per-note accuracy of CREPE median vs gold relative pitch,
  *using gold note boundaries* (isolates pitch errors from timing errors —
  today they're confounded).

Cache stems/pitch per song (they're expensive; results keyed like
batch_eval). Everything else in the strategy is gated on these numbers —
no more changing the pipeline on anecdote.

### Phase 1 — lyrics-first timing (replace the Whisper-driven merge)

When canonical lyrics exist (LRCLIB / pasted — the common case):

- **CTC forced alignment as the primary clock**: torchaudio's bundled
  `MMS_FA` (multilingual wav2vec2, ~20 ms frames, no new dependency) aligns
  the *full canonical text* to the separated vocal stem. Every canonical
  word gets a measured start/end — nothing skipped, elongated vowels get
  their true extent, phrase starts are acoustic, not attention artifacts.
  Implementation notes (investigated this session, unwritten):
  - lazy import + fail-soft per repo convention; CPU fallback works.
  - decode audio via `audio_io.load_mono` (ffmpeg), never torchaudio I/O.
  - chunk wav2vec2 emissions (~30 s windows), concat, single Viterbi
    (`torchaudio.functional.forced_align` / `bundle.get_aligner()`).
  - normalize tokens: NFD-strip accents, lowercase, a–z only; skip when
    <80 % of tokens romanize (ja/kana → keep current path).
  - keep per-**character** spans on each word dict (`char_spans`) —
    `assemble` cuts syllables at true acoustic boundaries instead of
    proportional spread (fixes elongation overlap directly).
  - gate on alignment score: below threshold (tune on Phase-0 data) fall
    back to current Whisper path.
- Whisper's remaining jobs: language detection, fallback when no canonical
  lyrics, and candidate text when LRCLIB misses.
- Expected to subsume this session's band-aids: `_snap_to_voiced` stays as
  a cheap refinement, but the count-in re-anchor and most of the gap-guard
  work becomes redundant on the FA path (keep them for the fallback path).

### Phase 2 — musical grid

Quantize onsets to the estimated musical beat (`tempo.estimate_bpm` →
sub-beat grid, e.g. 1/4 beat), writing the real BPM to `#BPM` like hand
charts do, instead of the fixed 240 grid. Gate on Phase-0 onset metrics
(quantization must not worsen onset error; it should improve *feel* and
gold-chart match since gold onsets sit on the same musical grid).

### Phase 3 — pitch on correct boundaries

With FA note boundaries, re-check pitch accuracy (Phase 0 pitch metric,
now unconfounded). Only then decide if CREPE needs help (median filter,
note-level Viterbi smoothing, or basic-pitch comparison).
